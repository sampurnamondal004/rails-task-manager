#Task Manager on Rails

A full-stack task management application with a **Ruby on Rails 8 API** backend and a **React** frontend, deployed on Render (API) and Vercel (frontend).

🔗 **Live Demo:** [rails-task-manager-omega.vercel.app](https://rails-task-manager-omega.vercel.app)

---

## Tech Stack

| Layer | Technology |
|---|---|
| Backend | Ruby on Rails 8.1.3 (API mode) |
| Language | Ruby 3.2.11 |
| Database (dev) | SQLite3 |
| Database (prod) | PostgreSQL |
| Web Server | Puma + Thruster |
| Background Jobs | Solid Queue |
| Caching | Solid Cache |
| WebSockets | Solid Cable |
| Frontend | React (Create React App) |
| CORS | rack-cors |
| Containerization | Docker |
| Deployment | Render (API) · Vercel (frontend) |

## Getting Started

### Prerequisites

- Ruby 3.2.11
- Bundler (`gem install bundler`)
- Node.js + npm (for the React frontend)
- SQLite3 (development)

### Backend Setup

```bash
# Clone the repository
git clone https://github.com/sampurnamondal004/rails-task-manager.git
cd rails-task-manager

# Install dependencies
bundle install

# Set up the database
bin/rails db:create db:migrate

# Start the Rails server
bin/rails server
```

The API will be available at `http://localhost:3000`.

### Frontend Setup

```bash
cd frontend

# Install dependencies
npm install

# Start the React development server
npm start
```

The frontend will be available at `http://localhost:3001` (or whichever port CRA assigns).

## Docker

A production-ready multi-stage Dockerfile is included. It uses Ruby 3.2.11 slim, jemalloc for reduced memory usage, and runs as a non-root user.

```bash
# Build the image
docker build -t task_manager .

# Run the container
docker run -d -p 80:80 \
  -e RAILS_MASTER_KEY=<value from config/master.key> \
  --name task_manager task_manager
```

## Deployment

### API — Render

The `render.yaml` file configures automatic deployment to [Render](https://render.com):

- **Build command:** `bundle install && bin/rails db:migrate`
- **Start command:** `bin/rails server -p $PORT -e production`
- **Environment variables required:**
  - `RAILS_MASTER_KEY` — copy from `config/master.key`
  - `SECRET_KEY_BASE` — auto-generated by Render

### Frontend — Vercel

The React frontend in `frontend/` is deployed to Vercel. Point the Vercel project root to the `frontend/` directory and use the default Create React App build settings (`npm run build`, output `build/`).

## Environment Variables

| Variable | Required | Description |
|---|---|---|
| `RAILS_MASTER_KEY` | Yes (prod) | Decrypts `config/credentials.yml.enc` |
| `SECRET_KEY_BASE` | Yes (prod) | Rails secret key; auto-generated on Render |
| `DATABASE_URL` | Yes (prod) | PostgreSQL connection string |
